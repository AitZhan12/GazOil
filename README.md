# АЗС — учёт смен (фронт + бэк)

Сервис оцифровки бумажного «Отчёта за смену»: ввод показаний колонок, разбивка
по типам оплаты, сверка кассы, месячный отчёт по операторам.

```
azs/
├─ azs-backend/                 Spring Boot REST API (Java 17, JPA, Flyway, PostgreSQL)
├─ Веб-админка учёта смен АЗС/  React + Vite + Tailwind/shadcn (фронт-админка)
├─ docker-compose.yml           PostgreSQL для разработки
└─ db-init.sql                  Создание роли/БД, если БД своя (не из docker)
```

## Как это поженено

Фронт больше не хранит данные в `localStorage` — он ходит в REST API бэка
(`src/app/lib/storage.ts` стал тонким HTTP-клиентом). Бэк — источник правды:
хранит сырые данные в PostgreSQL и возвращает их в той же форме, что и фронтовые
типы (`Operator`, `Shift`), включая рассчитанные поля (выручка, ожидаемая касса,
расхождение). Формулы расчёта на бэке (`ShiftCalculator`) повторяют один в один
фронтовые (`lib/calculations.ts`), поэтому превью в форме и сохранённые значения
совпадают.

### REST API
| Метод  | Путь                  | Назначение                |
|--------|-----------------------|---------------------------|
| GET    | `/api/operators`      | список операторов         |
| POST   | `/api/operators`      | создать оператора         |
| PUT    | `/api/operators/{id}` | изменить оператора        |
| GET    | `/api/shifts`         | список смен               |
| GET    | `/api/shifts/{id}`    | смена по id               |
| POST   | `/api/shifts`         | создать смену             |
| PUT    | `/api/shifts/{id}`    | изменить смену            |
| DELETE | `/api/shifts/{id}`    | удалить смену             |
| GET    | `/api/settings`       | ставки ЗП, цены, бонусы   |
| PUT    | `/api/settings`       | изменить настройки        |

Смена имеет **тип** (`full`=сутки / `day` / `night`). Зарплата за смену = ставка по
типу (настройки), бонус = ступень по литрам реализации; месячный итог суммирует это
по сменам. Все суммы и литры считаются до сотых. Контроль непрерывности счётчика:
форма предупреждает, если «начало» новой смены не совпало с «концом» прошлой.

## Запуск

### 1. PostgreSQL

Вариант А — Docker (чистая БД `azs/change_me`, порт 5432):
```bash
docker compose up -d        # или: docker-compose up -d
```
> Если 5432 уже занят локальным Postgres — освободите порт или используйте вариант Б.

Вариант Б — уже установленный PostgreSQL:
```bash
psql -U postgres -f db-init.sql
```
Креды берутся из `azs-backend/src/main/resources/application.yml`
(`azs` / `change_me` / `jdbc:postgresql://localhost:5432/azs`) — поправьте при необходимости.

### 2. Бэк
```bash
cd azs-backend
mvn spring-boot:run
```
Flyway сам накатит `V1` (создаст схему, станцию, 5 операторов, конфиг ЗП и таблицу бонусов). API на `http://localhost:8080`.

### 3. Фронт
```bash
cd "Веб-админка учёта смен АЗС"
npm install
npm run dev
```
Vite поднимет фронт (обычно `http://localhost:5173`). Адрес API задаётся в `.env`
(`VITE_API_URL`, по умолчанию `http://localhost:8080/api`).

## Проверка
```bash
curl http://localhost:8080/api/operators     # 5 стартовых операторов
```
Откройте фронт, создайте смену — она сохранится в PostgreSQL и появится в журнале,
месячный отчёт посчитается по данным из БД.
